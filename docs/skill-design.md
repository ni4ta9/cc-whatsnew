# whatsnew スキル設計ドキュメント

## 概要

`/whatsnew` は Claude Code の最新リリースノートを取得し、ユーザーの使用パターンを分析して
パーソナライズドレポート（MD + HTML）を生成するカスタムスキル。

## アーキテクチャ

### ファイル構成

```
skills/whatsnew/
├── SKILL.md                    # スキル定義・処理手順
└── assets/
    ├── template.ja.md          # 日本語 Markdown テンプレート
    ├── template.en.md          # 英語 Markdown テンプレート
    ├── template.ja.html        # 日本語 HTML テンプレート
    └── template.en.html        # 英語 HTML テンプレート
```

### 許可ツール

```
allowed-tools: Bash, Read, Write, Skill, AskUserQuestion
```

禁止ツール（WebFetch / Glob / Grep / Search）の詳細は SKILL.md を参照。

## 処理フロー

```
Step 1: 初期データ収集（並列）
  1a: 言語設定（settings.json）
  1b: プロファイル収集（report.html + AskUserQuestion）
  1c: CHANGELOG 取得（Bash curl + awk）
  1d: テンプレート読み込み + 出力ファイル確認（1a 完了後）

Step 2: コンテンツ分析（並列）
  2a: パーソナライズドレコメンデーション生成（1b + 1c 必要）
  2b: 最新バージョン詳細解説生成（1c 必要）

Step 3: 出力ファイル生成（並列）
  3a: MD ファイルを Write（LLM がテンプレート置換）
  3b: JSON データファイルを Write（HTML 用構造化データ）

Step 4: HTML 生成（Bash awk で JSON をテンプレートに注入）

Step 5: 完了報告（一時ファイル削除）+ セットアップ提案

Step 6: インタラクティブセットアップ（オプション）
  ユーザーが承諾した場合のみ実行
  おすすめ TOP N を1つずつ対話的に設定適用
  CLAUDE.md / settings.json に反映
```

### 依存関係図

```
1a ──────────────────────── 1d ─────────┐
1b ─┐                                   │
    ├─ 2a ─┐      ┌─ 3a ───────┐       │
1c ─┤      ├──────┤            ├── 5 ── 6（オプション）
    └─ 2b ─┘      └─ 3b ── 4 ─┘        │
                          ↑              │
                          └──────────────┘
```

1d は 1a の言語設定完了後に実行し、MD テンプレートを 3a に、HTML テンプレートパスを 4 に渡す。
Step 6 は Step 5 完了後にユーザーが承諾した場合のみ実行する。2a のおすすめリストを使用する。

## 設計上の重要な決定

### MD + HTML の両出力

MD は Git 管理・CLI での差分確認に適し、HTML はブラウザで視覚的にレポートを閲覧する用途。
両方を出力することで異なるユースケースに対応する。

### CHANGELOG は先頭3バージョンのみ取得

全 CHANGELOG は 129KB あり、コンテキストに全量を入れると処理が遅くなる。
直近3バージョンあればユーザーが把握すべき新機能は十分にカバーできるため、
`curl + awk` で先頭3バージョン分のみ抽出する。

### クライアントサイド JS による HTML 描画

LLM が HTML タグを直接生成すると、MD と合わせて同じコンテンツを2回フォーマット変換する必要があり、
出力トークン量が倍増して遅くなる。これを解消するため、HTML テンプレートにクライアントサイド JS レンダラーを内蔵した。

- LLM は JSON データ（~50行）を1回だけ Write する
- `awk` で JSON を HTML テンプレートの `__JSON_DATA__` プレースホルダーに注入
- ブラウザが HTML を開くと、テンプレート内の JS が JSON から pick-card / detail-card / release-section を描画

これにより LLM の出力トークン量が約50%削減される（MD 100行 + JSON 50行 vs 旧: MD 100行 + HTML 200行）。
外部ランタイム（Python / Node.js）に依存せず、`awk` のみで動作する。

### 既存ファイルの事前削除

Write ツールは既存ファイルがあると事前 Read が必要になり遅くなる。
上書き確認後に `rm -f` で事前削除し、新規作成として Write する。

### AskUserQuestion の省略禁止

Insight レポートの利用確認（1b）では、選択肢に「推奨」があると LLM が自動選択してしまう傾向がある。
「自動選択禁止・省略禁止」を明示して必ずユーザーに確認を求める。

### 言語別テンプレート

固定ラベル（セクションタイトル・URL テキスト等）を言語別ファイルにハードコードし、
動的プレースホルダーを最小化。LLM による翻訳揺れを防ぎ、出力の一貫性を確保する。
ユーザーカスタム HTML テンプレート（`~/.claude/whatsnew-template.html`）にも対応。

### インタラクティブセットアップ（Step 6）

レポート生成後に「おすすめ機能をそのまま設定に反映する」オプションを提供する。

- **オプトイン方式**: Step 5 の完了報告後に AskUserQuestion で提案し、ユーザーが承諾した場合のみ実行する
- **1件ずつ確認**: 各機能について適用/スキップ/終了を選べるようにし、ユーザーが制御権を持つ
- **適用先の自動判定**: 機能の種類（フック/許可コマンド/設定値/ワークフロー）に応じて settings.json か CLAUDE.md かを判定する
- **非破壊**: 既存設定は変更せず、追記のみ行う。既に同等の設定がある場合はスキップする
- **設定不要な機能の扱い**: スキルの活用提案など設定変更を伴わない機能は説明のみ表示し、適用確認をスキップする

### パーソナライズのマッチングロジック

おすすめ機能の選定は3段階の優先度で行う:
直接マッチ（使用中のツールに関係する機能） > ワークフロー補完（現在のワークフローを強化） > 未活用機能（まだ使っていない有用な機能）。
詳細は SKILL.md の Step 2a を参照。
